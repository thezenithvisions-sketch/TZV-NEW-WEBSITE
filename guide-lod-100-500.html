LOD 100–500 Quick Guide

A one-page reference to Level of Detail, what each level means, when to use it, and how to specify it so you never over- or under-model.

Download / Print Talk to our team

Level of Detail (LOD), sometimes called Level of Development, measures how developed and reliable a BIM element is, not how detailed it looks on screen. It is specified element by element and stage by stage. Use this table as a quick reference, then write LOD into your BIM Execution Plan.

LODMeansUse it for
100Conceptual, mass, area and orientation onlyFeasibility, early massing
200Approximate generic elements, rough size and locationSchematic design
300Precise geometry for coordinated documentationDesign development, CD set
350LOD 300 plus the interfaces needed for multidiscipline clash detectionCoordination between trades
400Detail, assembly and installation info sufficient to fabricateSteel, ductwork, precast fabrication
500Field-verified as-built geometry and dataHandover, facilities management

How to use LOD well

  • Match LOD to the project stage, not a single level for the whole job.
  • Specify LOD per element, structure, architecture, MEP can each differ.
  • Only push detail where a real downstream decision depends on it.
  • Capture it all in an LOD matrix inside your BIM Execution Plan.

The common mistakes

Asking for "LOD 400 throughout" because more sounds safer, it bloats the file, inflates the fee, and most of the detail is discarded when the design changes. The opposite, specifying LOD 200 then expecting shop-ready output, leaves the fabricator guessing and the rework lands on site. Right-size every element instead.
